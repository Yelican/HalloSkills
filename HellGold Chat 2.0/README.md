# HellGold Chat 2.0 · 黄金对话管理中心

**四合一 AI 对话分析看板** — 把每一次 Claude Code 对话变成可视化的认知报告。

---

## 这是什么

一个 Python 脚本，扫描你所有的 Claude Code 对话记录，生成一个**暗色科技风**的 HTML 看板，包含四个模块：

| 模块 | 解决什么问题 |
|------|------------|
| **💬 会话浏览** | "我上次跟 AI 聊了什么？" — 气泡式对话回放，搜索，筛选 |
| **💎 价值看板** | "这次聊得值不值？" — 5 级评估 + 内容分层 + 洞察提取 + 行动清单 |
| **🎯 提示词审计** | "我写的提示词好不好？" — 10 维评分 + 逐条分析 + 改写示例 |
| **🪞 批判镜** | "我有什么盲点？" — 尖锐的第三方视角，偏误检测 + 挑战问题 |

## 快速开始

```bash
# 1. 确保你的电脑有 Python 3
python --version

# 2. 运行脚本
python hellgold-chat-2.0.py

# 3. 打开桌面生成的 HTML 文件
# HellGold Chat 2.0.html
```

搞定。不需要网络，不需要安装任何依赖，双击 HTML 就用。

## 怎么工作的

1. 脚本扫描 `~/.claude/projects/` 目录（Claude Code 自动存储对话的地方）
2. 读取所有 JSONL 对话文件，提取用户消息和 AI 回复
3. 对每条对话做启发式分析（提示词质量评估、价值评分、内容分层、偏误检测）
4. 所有数据和可视化打包进一个自包含的 HTML 文件
5. 在浏览器中打开，交互式浏览和分析

## 和其他工具的关系

| 工具 | 关系 |
|------|------|
| **CC会话管理中心** | 同源数据（都读 `~/.claude/projects/`），但 HellGold Chat 增加了分析层 |
| **HalloChatGold** | HalloChatGold 是"手工单次复盘"，HellGold Chat 是"批量自动分析 + 可视化" |

**HellGold Chat 2.0 完全独立运行，不需要安装 CC会话管理中心或任何其他工具。**

## 界面说明

### 顶部导航
四个 Tab 切换：会话 → 价值 → 提示词 → 批判镜。右上角按钮切换明/暗主题。

### 左侧边栏
按项目分组的会话列表，支持搜索过滤。点击会话切换右侧内容。

### 💬 会话浏览
- **气泡式**对话展示：你的消息在右边（蓝紫渐变），AI 的在左边（暗色半透明）
- 默认**仅显示对话**（隐藏工具调用），可切换到"全部"或"仅工具"
- 点击气泡**展开/收起**完整内容
- 全局搜索：未选中会话时搜索所有会话的所有消息

### 💎 价值看板
- **Kirkpatrick 五级评估**：反应→学习→行为→成果→ROI，每项 0-2 分，展开看评分标准
- **评分详情**：每个维度可展开，显示三级评分标准 + 本对话证据 + 改进路径
- **复利评估**：杠杆率 / 半衰期 / 衍生价值，各有释义
- **DIKW 金字塔**：事实→方法→迁移→认知 四层占比
- **反事实价值**：VS 对比框 + 净增益 + 时间估算
- **核心洞察**：展开看来源事实 + 解释力 + 行动推演 + 可证伪挑战
- **RACI 清单**：彩色矩阵表格（R=执行者 A=负责人 C=顾问 I=知会方）

### 🎯 提示词审计
- **10 维评分体系**：目标清晰度、角色锚定、上下文充分性、约束条件、输出格式、示例质量、推理框架、语言精确度、安全公平、效率可维护
- 每项 1-5 分，带权重星标（★ = 重要性）
- 展开显示**改写示例**：❌ 你常写 / ✅ 应该写
- **逐条评分趋势**：本轮对话中各条提示词的质量变化
- **共性弱项**：得分最低的维度 + 具体改写方案

### 🪞 批判镜
- **总体判断**：基于对话深度的整体评价
- **核心盲点**：具体问题 + 详细分析 + 证据锚点
- **认知偏误检测**：确认偏误 / 可得性启发等
- **逻辑断裂点**：因果简化等推理漏洞
- **元认知挑战**：3-4 个让你自己反思的问题
- **改进路线图**：编号的、可操作的具体建议

## 配置自动更新

每次对话结束后自动更新看板，无需手动运行。

### 方法一：简单（保持脚本在桌面）

在 `~/.claude/settings.json` 的 `"hooks"` 字段中添加 `SessionEnd` 钩子：

```json
{
  "hooks": {
    "SessionEnd": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "python \"$HOME/Desktop/HellGold Chat 2.0/hellgold-chat-2.0.py\""
          }
        ]
      }
    ]
  }
}
```

路径中的空格需要用 `\"$HOME/...\"` 包裹。

### 方法二：干净（推荐，安装到 shared-scripts）

```bash
# 1. 把脚本复制到 shared-scripts（路径无空格，更稳定）
cp "Desktop/HellGold Chat 2.0/hellgold-chat-2.0.py" ~/.claude/shared-scripts/

# 2. 在 settings.json 中添加 hook
```

settings.json 配置：

```json
{
  "hooks": {
    "SessionEnd": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "python ~/.claude/shared-scripts/hellgold-chat-2.0.py"
          }
        ]
      }
    ]
  }
}
```

### 同时使用 CC会话管理中心 + HellGold Chat

如果你两个工具都要自动更新，在同一个 `SessionEnd` 钩子中添加两条命令：

```json
{
  "hooks": {
    "SessionEnd": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "python ~/.claude/shared-scripts/session/export-all-html.py"
          },
          {
            "type": "command",
            "command": "python \"$HOME/Desktop/HellGold Chat 2.0/hellgold-chat-2.0.py\""
          }
        ]
      }
    ]
  }
}
```

两条命令会在对话结束后**并行执行**，分别生成 CC会话管理中心.html 和 HellGold Chat 2.0.html，互不干扰。

## 常见问题

**Q: 为什么打开 HTML 是空白的？**
A: 极少数情况是对话数据太大导致浏览器卡住。刷新页面试试。这个版本的 HTML 没有任何外部依赖，不会因为 CDN 加载慢而白屏。

**Q: 分析数据准确吗？**
A: 当前版本使用启发式算法（规则 + 统计），不是真正的 LLM 分析。它模拟了一个分析师的判断逻辑，给出"方向性正确"的评分。真正的 LLM 接入版本在路线图中。

**Q: 我的对话是隐私的，安全吗？**
A: 所有数据都在你的电脑本地处理，HTML 文件也是纯前端的，没有任何数据离开你的电脑。脚本只读取 `~/.claude/projects/` 下的 JSONL 文件，不联网传输任何内容。

**Q: 能支持 ChatGPT / DeepSeek 的对话吗？**
A: 当前版本专门为 Claude Code 的 JSONL 格式设计。如果你能把其他平台的对话导出为类似格式，理论上也可以。欢迎 PR。
