# HalloChatGold — 对话淘金术

**把任何讨论变成认知资产。** 基于 6 个被验证的思维模型的有机结合，形成一套 7 步流水线，让每次对话都能快速提炼出洞察、行动和长期价值。

> 想了解这 6 个模型是如何被验证、筛选、最终融合成这套方法的？ → [BIRTH.md](./BIRTH.md)

| 步骤 | 模型名称 | 学术/业界来源 | 验证状态 |

## 7 步管道详解

### 第 1 步 · 收集（Collect）— 原始材料

**如果用户提供文本/截图：** 直接读取并分段。
**如果是刚发生的对话：** 基于当前会话上下文回顾主要内容。
**如果是外部聊天：** 让用户粘贴文本（微信截图需要用户先转文字）。

输出：讨论主题 + 参与方 + 时长/篇幅概要。

---

### 第 2 步 · 过滤（Filter）— 4层价值漏斗

> 来源：DIKW金字塔 (Ackoff 1989)。90% 的讨论价值集中在最上层的 10% 内容里。

**内容分层就是帮你分清「哪些值得反复看，哪些扫一眼就够」。**
一次对话的内容可以拆成四层，越往上越值钱——底层是操作，顶层是认知。

| 层级 | 定义 | 讨论中的对应内容 | 占比参考 |
|------|------|----------------|---------|
| **L1 事实信息** | 客观的、可查的操作性知识 | 命令、链接、配置参数、具体数字 | ~10% |
| **L2 方法技能** | 解决某一类问题的通用步骤 | 工作流、配置方法、操作流程 | ~20% |
| **L3 能力迁移** | 能用到其他领域的通用能力 | 工具选型方法论、决策框架 | ~30% |
| **L4 底层认知** | 改变决策方式的底层规律 | "集成度与能力成反比"类规律 | ~40% |

**操作：**
- 把讨论内容逐条扔进对应层级
- L1 只保留关键链接和命令（放附录）
- **重点整理 L3 和 L4**

**分层结论怎么看：**
- L3+L4 占比越高 → 这次讨论越有深度，值得反复消化
- L1 占比高 → 偏操作型，实用但保质期短，提取关键信息就走

**分类质量检查清单（分类时逐条核实）：**
- [ ] L1：这段内容可否被独立验证？（有具体实体/数字/可查证事实）
- [ ] L2：这段内容能否指导"怎么做"？（步骤可重复，别人按描述操作可得类似结果）
- [ ] L3：这个能力脱离原领域后是否仍成立？（举至少2个不同领域）
- [ ] L4：这个认知是否动摇了某个原有假设？（追问"所以呢"能落地到行动）

**分类红线（触碰任意一条，强制降一级）：**
- 这段内容降到下一级也不损失价值 → 它就是下一级
- 能在百度百科/教科书找到相同表述 → 不是高价值洞见
- 纯感性共鸣被当成认知价值 → 去掉情绪再看还剩什么
- 用了"底层逻辑""本质""第一性原理"包装常识 → 去掉这些词重新判断

### 第 3 步 · 评估（Evaluate）— Kirkpatrick 五级评估

> 来源：Kirkpatrick 四级 (1959) + Phillips ROI (1996)。评估单次讨论的投入产出比。

**五级评估从「即时反应」一直穿透到「长期回报」，给你的讨论打个分。**

**评分纪律（所有评分步骤通用）：**
1. **从 0 起评** — 每项起始分 = 0。加 1 分必须有对话原文作为证据支撑，不接受"我感觉""大概有"类推断。
2. **证据锚定** — 每加 1 分必须附一句引文。找不到引文 = 不能加分。
3. **扣分不可跳过** — 每个扣分项独立检查，即使给了满分也必须过一遍。
4. **疑罪从低** — 不确定属于哪个分数等级时，选较低的。
5. **审计触发** — 5 个层级中有 4 个以上满分（2分）→ 自动审计，AI 须逐项提供反驳证据，否则降 1 分。

| 级别 | 评分 | 判定标准（满足全部条件才给分） | 扣分项 |
|------|------|----------------------------|--------|
| **L1 反应** | 0 | 用户只发了1条消息/全程只回"嗯""好的"等无信息量词/未提出任何追问 | — |
| | 1 | 用户对对话的具体部分做出评价（非笼统好评）/主动提出至少1个追问/表达了具体情绪 | -1：好评全是笼统的（"很好""有收获"）无具体指向；-1：AI引导多次后才回应的非自发参与；-1：说"收获很大"但说不出具体收获 |
| | 2 | 用户能追溯到具体触发节点（"你说到xxx时我突然意识到…"）/主动将话题延伸到自身实际情况/未等AI提问就主动分享关联经历 | 最终分≤1：用户未主动发起任何话题或追问 |
| **L2 学习** | 0 | 用户全程仅复述已知信息/对话无深度处理痕迹 | — |
| | 1 | 用户用自己的话复述了新概念/将新信息与已有知识建立了联系/提出了"加工后"的问题（"如果X成立，那么Y会…"）/纠正了自己先前的错误认知 | -1："原来如此"式被动接收不算学习；-1：用户说不清新旧认知的区别；-2：学习内容完全由AI直接喂给无加工 |
| | 2 | 用户描述了思维模型的转变（"我以前用A，现在知道该用B"）/将新知识独立迁移到对话未涉及的领域/用费曼法把新概念解释给第三方 | 最终分≤1：学习内容是单一孤立知识点未被整合到更大框架 |
| **L3 行为** | 0 | 用户未提及任何与自身实际的关联/对话停留在纯理论层/未表达应用意图 | — |
| | 1 | 用户明确指出了一处具体适用场景（时间+情境）/描述了计划中的行为改变/提问方向是"如何做"而非"是什么" | -1：行为改变停留在愿望层无具体路径（"以后注意"）；-1：改变明显不切实际；-1：知道该做什么但无行动动机 |
| | 2 | 用户说清了"①改什么 ②什么情境下改 ③怎么验证改有效"/识别出跨场景迁移原则并举例至少2个不同场景/主动承诺了问责机制（"我三天后把结果发你"） | 最终分≤1：用户未说明至少一个具体实施场景 |
| **L4 成果** | 0 | 对话结束时无任何可测量的产出/成果是纯粹内在的（"我有启发"）无外部可验证信号 | — |
| | 1 | 用户作出了具体决定或改变了原有决定/形成了包含至少1个有形步骤的行动方案/设定了可衡量的成功标准 | -1：行动方案仅有意图无时间线；-2：成果依赖用户无法控制的外部因素 |
| | 2 | 行动方案满足SMART标准（具体+可衡量+可达成+相关+截止时间）/产出可由第三方验证/或对话直接产出了有形交付物（文档、框架图、模板等） | 最终分≤1：用户未做出任何可核实的承诺（具体时间/地点/产出） |
| **L5 ROI** | 0 | 对话价值仅限于对话本身/使用是一次性/用户未表达延续使用的意愿 | — |
| | 1 | 用户表示对话影响了一个重复性决策（"以后每次都用"）/请求保存/回看/转发对话内容/识别了价值如何延续到未来 | -1：长期价值只是乐观预期无作用机制；-2：计划"分享给他人"但说不出具体人名或场景；-1：ROI完全抽象（"提升认知"）无有形应用 |
| | 2 | 用户描述了价值的复利效应（"用一次→条件更好→下次价值更大"）/对话产生了可反复使用的资产（框架/流程/模板）/用户计划将见解传播给具体对象或群体 | 最终分≤1：用户只消费了内容无任何生产性产出 |

**总分换算：**
- 10分：五级全2分，无扣分 → 殿堂级对话
- 8-9分：多为2分，个别1分 → 高价值
- 5-7分：多为1分 → 中等价值
- 1-4分：出现0分 → 低价值

**评估结论怎么看：**
- 总分决定价值层级，但**单项低分和扣分项才是改进方向**
- 连续3次同一维度低分 → 那个维度是你的盲区，下次聊之前先想好怎么补
- 扣分项比分数本身更值得关注——它告诉你这次讨论哪里没做好

### 第 4 步 · 提炼（Extract）— 3级洞察金字塔

> 来源：Peirce 符号学 → 批判性思维层级。事实会过时，观点有局限，洞察永不过时。

**操作流程：**

1. **Facts 提取** — 列出讨论中所有客观可验证的信息点
2. **Opinions 提取** — 从 Facts 中输出主观判断和经验总结
3. **Insights 提炼** — 追问："这个观点背后的底层规律是什么？它还能解释什么其他现象？"

**输出格式：**

> ### 💎 核心洞察
>
> **洞察 1：[一句话]**
> - 来源事实：[哪些讨论内容支撑这个洞察]
> - 解释力：[为什么你考研能坚持但健身不能？还能解释什么其他现象]
> - 行动推演：[如果要验证这个洞察，明天可以做什么具体动作]
> - 关联挑战：[这个洞察和「XX」矛盾吗？什么情况下它不成立]
>
> **洞察 2：...**

**洞见质量标准：**
- 上限 7 条，不设下限，**可少不可滥** — 宁可只有 2 条精的，不要硬凑 7 条水的
- 必须经过三轮筛查才能入选：① 有事实支撑？② 能解释 2 个以上无关现象？③ 有可证伪性？
- 每条洞察必须能展开 200 字以上的实质性思考（不够展开的就合并或删掉）
- 重点不是"对不对"，是"有没有用" — 能不能指导行动或改变认知

**洞察质量检查清单：**
- [ ] **可证伪性检验** — 这个洞察在什么情况下不成立？说不出来 → 太虚，降级
- [ ] **丰产性检验** — 这个洞察让你想继续追问什么？（"那是不是也意味着…"）问不出来 → 太平，降级
- [ ] **肥皂剧测试** — 这个洞察能出现在肥皂剧台词里吗？（"人总是伤害自己最亲近的人"）能 → 太浅，降级
- [ ] **跨领域强制测试** — 这个解释至少还能用在2个不同领域？不够 → 降级
- [ ] **术语剥离测试** — 去掉"底层逻辑""认知升级""本质"这些词，还剩什么实质内容？

**洞察结论怎么看：**
- 好的洞察每条都值得你停下来想 5 分钟——它解释了过去，也指向了未来
- 如果看完觉得「有道理但不知道然后呢」→ 说明这条洞察还没挖到底，继续追问「那我能做什么」

**金句收尾：**
所有洞察输出完后，追加一句与主题高度相关的名言/格言，并简短联系本次核心洞察做延展。不用解释金句本身，而是用它照亮讨论中的一个侧面。宁缺毋滥，没有合适的就不加。

### 第 5 步 · 验证（Verify）— 反事实价值模型

> 来源：Pearl 反事实因果推理 (2018)。避免"理所当然"错觉，真正测量讨论的不可替代性。

**反事实验证就是问自己：如果没聊这次，我会亏多少？**
听起来扎心，但只有这么问才能挤出讨论的真实价值。

**灵魂三问：**

| 问题 | 用意 |
|------|------|
| "如果今天没有这次讨论，我会怎么做？" | 测量替代路径的成本 |
| "如果今天没有这次讨论，我会损失什么？" | 量化讨论的净价值 |
| "为了获得这次讨论的价值，我愿意付多少钱？" | 把模糊价值转成具体数字 |

**输出：** 一个反事实判断——"这次讨论替我节省了 X 天摸索时间，避免了 Y 个坑，价值至少 Z 元。"

**验证结论怎么看：**
- **节省了时间** → 这次讨论帮你走了捷径
- **避免了坑** → 价值比表面数字更大（没发生的损失最难被感知）
- **愿意付钱** → 绝对值越高，说明这次讨论对你越不可替代

**反事实质量检查：**
- [ ] **替代路径是否具体？** — 不是"会更迷茫"，而是"会去B站搜XX关键词看3个视频"
- [ ] **是否有成本估算？** — 时间/金钱/质量对比，三者至少有一个
- [ ] **三问自洽检验** — 替代路径成本 > 价值货币化 > 净价值。不自洽 → 重评
- [ ] **时间拉伸测试** — 一个月后这次讨论还剩下什么？一年后呢？

**反事实红线：**
- "无价""极其珍贵"类表述 → **扣分信号**，不是加分信号
- Q1说"什么也得不到"但Q3只给50块 → 逻辑矛盾，重新评估
- 情感价值覆盖了认知价值 → 要求分别打分（认知价值/情感价值/激励价值）

### 第 6 步 · 落地（Act）— RACI 行动矩阵

> 来源：PMI 项目管理标准 (1960s起)。没转化的讨论价值为 0。

**RACI 是个责任分配工具，帮你搞清楚：谁干活、谁背锅、问谁意见、告诉谁结果。**
一次高质量讨论的价值，最终体现在你接下来做的事上。

| 角色 | 定义 | 行动项 | 截止时间 |
|------|------|--------|---------|
| **R（执行者）** | 具体干活的人（就是你） | [具体行动] | [时间] |
| **A（问责人）** | 对最终结果负责的人（通常也是你） | — | — |
| **C（顾问）** | 需要问一嘴意见的人/资源 | — | — |
| **I（告知对象）** | 需要知会一声的人 | — | — |

> 标准 PMI 定义：R = 干活的人，A = 对最终交付质量负责的人（通常 R 和 A 是同一个人），C = 提供输入的人，I = 只需知道结果的人。A 不是"批准"，是"问责"。

**所以这次讨论之后：**
1. 先把 R 列的事排进日程
2. 如果需要别人配合 → 填 C（什么时候问谁）
3. 如果有需要同步的人 → 填 I（什么时候告知谁）
4. 至少 1 个 R 类行动，每个必须有明确截止时间

**行动项质量检查清单：**
- [ ] 有明确的 R（具体到人名，不是"团队""大家"）
- [ ] 有且仅有 1 个 A（PMI 铁律：每任务 1 个 A，2 个 A = 没人真正负责）
- [ ] 行动描述用动词而非抽象名词（"写完提纲"vs"好好准备"）
- [ ] 有具体截止时间（YYYY-MM-DD，不是"尽快""下次"）
- [ ] 有可验收的完成标准（比如"输出是一份3页以内的摘要"）
- [ ] C（顾问）不超过 3 个，I（告知对象）不遗漏关键人

**周一早测试**：假设周一早上你拿到这个行动项，能在周五前独立完成吗？如果不能——因为要等别人、因为没想清楚怎么做、因为不知道什么算"完成"——说明行动项缺要素，需要补。

**行动红线：**
- 截止时间缺失 → 最高评 1 分（没有时间 = 不会做）
- 无 A（问责人）→ 最高评 1 分（没人最终负责）
- 同时缺时间和 A → 这不是行动，是愿望清单

### 第 7 步 · 预测（Forecast）— 复利效应评估

> 来源：Naval 杠杆率 + Samuel Arbesman 知识半衰期 (2012) + 网络效应衍生价值。

**复利效应，简单说就是：这次讨论的价值是「用完就废」还是「越滚越大」？**

> 评分标准从 1-10 改为 0-2 分，与 Kirkpatrick 全篇统一。评分纪律见第 3 步。

#### 杠杆率（学会 1 个方法能解决多少问题）

| 评分 | 判定标准 | 扣分项 |
|------|---------|--------|
| 0 | 方法/见解只能解决它针对的那一个具体问题；使用是一次性消耗 | — |
| 1 | 用户识别了至少1个额外的可应用场景（同领域不同问题）；方法经小幅调整可适用相似场景 | -1：杠杆率只是推测无实际场景；-1：依赖理想条件而用户实际不满足 |
| 2 | 方法是元技能或通用框架可跨领域使用；一次投入产生持续多领域收益；用户识别了至少3个不同场景 | 最终分≤1：用户只在被动追问下才说出其他场景 |

#### 半衰期（这次讨论的价值能持续多久）

| 评分 | 判定标准 | 扣分项 |
|------|---------|--------|
| 0 | 见解有明显时效性（针对具体事件/版本/趋势）；价值一次使用后即大幅衰减 | — |
| 1 | 核心原理在具体案例过时后依然成立；方法在数周到数月内有稳定价值不衰减 | -1：见解绑定特定工具/平台/软件版本；-2：自认为"有持久价值"但其实是老生常谈套话 |
| 2 | 见解是根本性的——即使环境大幅变化也成立；使用越多价值越大（学习曲线效应）；是"如何思考"而非"思考什么"（元认知层） | 最终分≤1：见解一句话就能说完（缺乏可展开的深度） |

#### 衍生价值（能带来多少其他机会）

| 评分 | 判定标准 | 扣分项 |
|------|---------|--------|
| 0 | 对话未产生任何新问题/连接/探索方向；价值严格等于对话边界不溢出 | — |
| 1 | 对话激发了至少1个新想法或新方向；参与者连接了两个先前无关的知识领域 | -1："新机会"只是原问题的另一种表述；-2：衍生结果实际是认知负担而非清晰 |
| 2 | 参与者列出至少2个具体后续行动方向；衍生价值是自我加强的（一个洞察引出另一个）；对话改变了参与者学习和解决问题的方式本身 | 最终分≤1："衍生价值"完全在AI输出里，参与者无任何延伸响应 |

**综合评级：**
- S 级：三项均 ≥2，且无扣分项执行 → "改变轨迹的对话"
- A 级：三项均 ≥1，或总分 ≥4/6
- B 级：总分 ≥2/6
- C 级：总分 <2/6，或任意一项 =0

**评级留言：**
评级不只是打分，是告诉你该怎么对待这次讨论的成果——

| 评级 | 它意味着 | 你该怎么做 |
|------|---------|-----------|
| S | 这次讨论可能改变你的轨迹 | 整理成笔记，纳入核心框架，定期回顾 |
| A | 长期价值明确，值得投入 | 一周内把行动落地，让复利转起来 |
| B | 有收获但深度不够 | 下次围绕同一话题再聊一次，挖深一层 |
| C | 当下有用，但别当宝贝 | 提取可用信息就走，别花时间反复看 |

## 执行原则

1. **不装腔** — 洞察必须能在现实中验证，不说"赋能""底层逻辑"之类的空词
2. **洞察上限 7 条，但核心是「可少不可滥」** — 每次入选都要经过"有事实吗→能解释别的吗→有反例吗"三轮筛查，凑不齐就别硬凑
3. **每条行动必须有截止时间** — 没有时间 = 不会做
4. **反事实必须具体** — 不能是"我会更迷茫"这种废话，要说出具体替代路径
5. **用户说够了就停** — 别为了走完 7 步而走 7 步

## 视角规则（重要）

这份报告是给**讨论的参与者本人**看的，不是写给别人看的第三者报告。

- **正文全程用「我」的视角**，或者按中文习惯省略主语
- **禁止用「用户」称呼使用者**，也不要用 AI 的人称来写这篇报告
- 报告应该像是参与者自己复盘写的，不是 AI 在评价他
- 如果 AI 有人设（取了名字），可以在文末加一行 `编辑：XXX / AI 辅助生成`，但正文不能变成 AI 视角

**正例：**
```

```

**反例：**

### 自动存档规则（v2.0 — 看板集成）

自动存档的目的是把每次分析的结构化数据积累起来，让 `build.py` 聚合成统一看板。
存档是一个独立步骤——即使存档失败，也不影响主报告的生成和导出。
